# @visualdsa/db — Prisma 7 + PostgreSQL

Owns the database schema and the generated Prisma client. Consumed by the API
(`PrismaService`). Uses the **`@prisma/adapter-pg` driver adapter** (no query
engine binary).

## Layout

- `prisma/schema/` — **multi-file schema** (Prisma 7). `schema.prisma` holds the
  generator (`provider = "prisma-client"`, `output = "../generated"`) + datasource.
  Models/enums live in sibling files (e.g. `user.prisma`), merged by Prisma.
- `prisma/generated/` — **generated client (gitignored)**. `client.ts` is the
  entry; `enums.ts`, `models/`, `internal/` support it. Regenerated by
  `pnpm db:generate` (or `pnpm codegen`).
- `prisma/migrations/` — migration history.
- `prisma.config.ts` — Prisma 7 `defineConfig` (schema dir, migrations path,
  `datasource.url` from `DATABASE_URL` via `dotenv`).
- `src/index.ts` — re-exports `PrismaClient` + types from `prisma/generated`.
- `tsup.config.ts` — builds dual CJS/ESM.

## Adding / changing a model

1. Edit a file under `prisma/schema/` (or add a new `<name>.prisma`).
2. `pnpm --filter @visualdsa/db run db:generate` — regenerate the client.
3. `pnpm --filter @visualdsa/db run db:push` (dev, no migration) or
   `db:migrate` (creates a migration). Note: the existing migration is named
   `20260629091832_inti_db` (typo "inti") — future migrations should be named
   properly.
4. `pnpm --filter @visualdsa/db run build` — rebuild `dist/` (the API consumes
   `dist/index.js`).

## Critical gotcha — `import.meta.url` under CJS

`prisma/generated/client.ts` runs
`globalThis['__dirname'] = path.dirname(fileURLToPath(import.meta.url))` at module
top level. esbuild stubs `import.meta` → `{}` under CJS, so this throws on import.
`tsup.config.ts` injects a **format-aware** banner (`__import_meta_url` via
`pathToFileURL(__filename)`) only for the CJS build — ESM has `import.meta.url`
natively. **Do not remove that banner**, or importing `@visualdsa/db` from the
CJS NestJS app crashes at boot.

## Commands

```
pnpm db:generate   # regenerate client
pnpm db:push       # schema -> DB (dev)
pnpm db:migrate    # create + apply migration
pnpm db:deploy     # apply migrations (prod)
pnpm db:studio     # prisma studio
```

`DATABASE_URL` is read from `packages/db/.env` (copied from `.env.example`).
